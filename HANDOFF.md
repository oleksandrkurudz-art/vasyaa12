## 3. Критичні «граблі» (коштували дебагу, в коді неочевидно)

- **Vercel ріже тіло запиту ~4.5 МБ** → фото з телефона (5-8 МБ) падало з **413** ще до сервера.
  Тому стиснення **у браузері** перед відправкою (`src/components/admin/CoverUpload.tsx`,
  canvas→WebP, 2048px/q0.9). `serverActions.bodySizeLimit` тут НЕ допомагає.
- **`sharp` — нативний модуль**, Vercel не клав його в бандл → 500. Фікс у `next.config`:
  `outputFileTracingIncludes: { "/admin/**": ["node_modules/sharp/**/*", "node_modules/@img/**/*"] }`.
  Сервер чіпає лише сирий JPEG/PNG; WebP від браузера **не перекодовує** (інакше подвійне стиснення).
- **Стейл Prisma-клієнт** мовчки давав 404 на розділах/статтях. Фікс: скрипт `predev` →
  `prisma generate` на кожному `npm run dev`. Запускаєш dev інакше — зроби `npx prisma generate` сам.
- **Пароль БД містить `@`** → у `DATABASE_URL` кодувати як `%40`.
- **Пул з'єднань: Transaction pooler (порт 6543), НЕ Session pooler (5432).** Session-режим тримав
  з'єднання на весь час сесії → локальний dev + прод-лямбди перебивали ліміт (~15) і давали
  `EMAXCONNSESSION`. Transaction pooler мультиплексує короткі з'єднання. Адаптер `PrismaPg` за
  замовчуванням НЕ кешує prepared statements, тож із transaction-режимом сумісний без `pgbouncer=true`.
  Перехід = лише порт `5432→6543` у `DATABASE_URL` (хост/юзер/пароль ті самі) локально **і** у Vercel env.
- **Деплой вручну:** `vercel deploy --prod --yes` (CLI залогінений як `oleksandrkurudz-art`).
  Env: `printf '%s' '<val>' | vercel env add <NAME> production`. ⚠️ Локальний `main` може бути
  попереду GitHub — push не автоматичний.
- **Схемою керуємо через Supabase MCP** (`apply_migration`/`execute_sql`), build НЕ робить
  `migrate deploy`. Нова міграція: `npx prisma migrate diff … --script` → застосувати через MCP.
  MCP не віддає пароль БД і service-role ключ — їх дає користувач.
- **`preview_screenshot` зависає** (30s) на завантаженні фото з picsum. Перевіряй через
  `preview_eval`/`preview_inspect` (DOM, computed styles, статуси), не скріншотами.
- **Не чисти `.next` / не запускай прод-білд, поки крутиться dev** — ENOENT/MODULE_NOT_FOUND.
- **API тривог (`airraid.ts`) жорстко лімітує** (429 при частих запитах) — тому власний кеш 60с;
  область зашита як `AIR_RAID_REGION = "Івано-Франківська область"` (точна назва-ключ з API).

### Env (Vercel Production + локальний `.env` у gitignore)
`DATABASE_URL`, `ADMIN_PASSWORD`, `SESSION_SECRET`, `SUPABASE_URL`,
`SUPABASE_SERVICE_ROLE_KEY` (формат `sb_secret_…`, лише сервер). У репо секретів немає.
Локальний `npm run dev` ходить у ту саму прод-базу Supabase.

## 4. Дизайн (вектор + ключові рішення)

Стиль ТСН: темна шапка (`Header.tsx`, `neutral-900`, не липка) + окрема **липка** смуга меню
(`NavBar.tsx`, `neutral-100`, синя акцент-лінія). Герой (`HeroBlock.tsx`) — full-bleed фонове
фото головної новини. Палітра: `brand-600 = #2563eb` (синій акцент). Шрифти: **Inter** (текст)
+ **Unbounded** (`font-display`, заголовки/логотип; повна кирилиця, через `next/font/google`).
⚠️ Не називай токен `@theme` як `--font-display` — він конфліктує з кастомним класом `.font-display`
(заголовки тоді мовчки лишаються Inter). Токен лишай під іншим імʼям; клас бере `var(--font-unbounded)`.

### ⚠️ Ширина (важливо для нових публічних сторінок!)
`<main>` у `(public)/layout.tsx` **не має `max-w-7xl`** (щоб герой був full-bleed без `100vw`,
який давав горизонтальний скрол). **Кожна публічна сторінка сама обгортає контент** у
`<div className="mx-auto w-full max-w-7xl px-4 py-6">`.

### Дрібниці, які легко зламати
- Перегляди показуються **лише коли > 100** (хелпер `hasEnoughViews` у `format.ts`).
- Колірні класи розділів (9 шт.) — **літеральні Tailwind-рядки** в `CATEGORY_STYLES` (щоб сканер їх згенерував).
- **«Терміново» ПОВЕРНУЛИ** (`BreakingBar.tsx`): авто-тривога (пріоритет) або ручна `breaking`-новина.
- Громади: новина з `communityId = null` — загальнорайонна (видно в усіх громадах); вибір громади в cookie `community`.
- Головна: текстова `NewsTicker` (ліворуч на ПК — «Головні події» з фото, `hidden lg:block`).
- **Категорії й громади живуть у БД** — нову треба додати І в `categories.ts`/`communities.ts`, І в БД (через Supabase MCP).
